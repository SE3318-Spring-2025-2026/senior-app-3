Process 7.4 Pipeline (10 Steps):
1. Input Validation: Auth + sprint exists
2. Sprint Lock Check: Prevent changes after deadline (409)
3. Fetch Members: All approved group members
4. Load Targets: D8 SprintTarget configuration
5. Calculate Total: Sum of completed story points
6. Compute Ratios: Per-student ratio = completed / target
7. Format Output: Round to 4 decimal places
8. Atomic Update: MongoDB transaction (all-or-nothing)
9. Update Timestamp: recalculatedAt in SprintRecord
10. Return Summary: Per-student breakdown + metadata
